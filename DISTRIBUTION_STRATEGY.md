## Platform Roles & Responsibilities

### 1. OSF (Open Science Framework) вҖ” Project Hub
**Role**: The front door. One OSF project page that links to everything else.

**What goes here**:
- **Project overview** вҖ” research program description, 6-paper arc, dependency DAG
- **Preprints** вҖ” PDF + TeX for ALL papers (Paper 1 through 6, plus BENI data paper)
- **Supplementary materials** вҖ” all figures, annotation protocols, dataset cards
- **Links to** вҖ” GitHub repo, Zenodo DOIs, Hugging Face models, Mendeley Data

**Structure**:
```
BENI Research Program (OSF Project)
в”ңв”Җв”Җ Components:

в”Ӯ   в”ңв”Җв”Җ Paper 2: Systematic Review (preprint)
```

## Citation Funnel Flow

### How a Researcher Discovers the Work

**Path A (NLP researcher)**:
```
Hugging Face (discovers BENI model)
  вҶ’ Reads model card вҶ’ cites Paper 3
  вҶ’ Follows link to GitHub вҶ’ runs code
  вҶ’ Follows link to OSF вҶ’ reads preprints
  вҶ’ Finds Zenodo DOI вҶ’ cites BENI dataset
```

**Path B (Economist)**:
```
OSF (discovers the research program)
  вҶ’ Reads Paper 4 (nowcasting) abstract
  вҶ’ Downloads preprint PDF
  вҶ’ Follows link to GitHub вҶ’ runs nowcasting
  вҶ’ Finds Mendeley DOI вҶ’ cites Potrika corpus
```

**Path C (Social scientist)**:
```
OSF (discovers Paper 5 survey)
  вҶ’ Reads the 110-year taxonomy
  вҶ’ Discovers BENI as case study
  вҶ’ Follows link to Paper 3/4/6
  вҶ’ Cites Paper 5 as "default entry point"
```

### How Citations Flow

```
Paper 5 (survey) вҶҗ cited by anyone writing "related work"
   в”Ӯ
   в”ңв”Җв”Җ cites Paper 2 (systematic review) вҶҗ the ENI review
   в”Ӯ
   в”ңв”Җв”Җ cites Paper 3 (BENI pipeline) вҶҗ the measurement tool
   в”Ӯ
   в”ңв”Җв”Җ cites Paper 4 (nowcasting) вҶҗ the application
   в”Ӯ
   в””в”Җв”Җ cites Paper 6 (LLM extraction) вҶҗ the frontier method

BENI v1.0 Data Paper вҶҗ cited by Papers 3, 4, 6 as data source
Potrika Corpus (Mendeley) вҶҗ cited by BENI Data Paper
```

## Key Principles

1. **OSF is the hub** вҖ” every other platform links TO it, and it links TO every other platform
2. **Don't duplicate large files** вҖ” raw corpus only on Mendeley; models only on HF; datasets on Zenodo
3. **Every platform gets a CITATION.cff** вҖ” researchers can cite from wherever they discover
4. **Badges in GitHub README** вҖ” Zenodo "DOI", OSF "View on OSF", HF "Model on HF", Mendeley "Dataset"
5. **Version everything** вҖ” BENI v1.0, v1.1, etc. Use semantic versioning for dataset releases
6. **The funnel works because everything connects** вҖ” no orphan platforms
